Github Forks and Pull request


Fork is basically "clone" an entire branch from one Github user to another


Very usefull if you want to experiment in a branc but not affect the original branch


if you like the experiments and you want to merge them back together, you are using the pull request


Owner of the original branch has to accept the pull request


to create a pull request press edit or the fork button on the screen in the project.


Note: if you are working in a branch, it will commit changes to main branch in fork user's account
