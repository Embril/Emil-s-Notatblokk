Terms

Repository is a repository of "files"


Commit is equal to Save in office


Commit Hash is the identifierf for a commit


branches if you want to commit a change but not sure if you wll keep the change, you can "branch" off


Pull request and a merge. Push and pull is important in Github. it lets other users request to push or pull branches into the main branch. (merge)


Fork: to "clone" an entire branch from anoter user to make changes. This wll not commit in the original branch.


pull request: you create a pull request when working in a fork and want to merge the branch back to the original. Basically you are creating a push request, however, because the original user has to accept the push its called a pull request.
